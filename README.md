# DesignPatternsThePainfulWay


## Idea
Aprender a usar los patrones de Diseño a partir de sufrir el desconocimietno. 

Para esto, vamos a hacer tres cosas:
Parte 1) Proponer un problema simple, para modelar y escribir en codigo (o pseudo codigo) de cualquier manera que salga natural. Sin patrones requeridos.

Parte 2) Voy a agregar un requerimiento que desafia la solucion anterior, forzando cadenas de if/else, duplicacion de codigo, subclases exponenciales, etc.

Parte 3) Proponer la solucion "elegante" - usando patrones de diseño y generando el diagrama de clase corrrespondiente.

## Enfoque
- Java como lenguaje principal.
- La documentación (enunciados y respuestas) está en [/docs](docs/README.md).
- Una propuesta de solución por cada parte.

  
o algo asi.
